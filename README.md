# 蝦皮分潤後台 Dashboard

最新版本：`v0.5`

更新日期：`2026-07-16`

這是一個輕量的蝦皮分潤追蹤後台，用來查看每月分潤、訂單、點擊、商品成效與待處理行動。第一階段目標是追蹤「每月 NT$5,000 蝦皮分潤」，讓選品、發文與成效優化可以在同一個畫面完成判讀。

## 目前內容

- `index.html`：Dashboard 畫面與區塊結構
- `styles.css`：桌面版、平板與手機版的響應式樣式
- `app.js`：CSV 讀取、資料正規化、欄位別名、指標計算、推薦規則與圖表繪製
- `dashboard_spec_for_da-bing-nai.md`：v0.5 Dashboard 規格文件

## 使用方式

直接開啟 `index.html` 即可使用。

畫面會先嘗試讀取設定好的 Google Sheet CSV。若試算表尚未發布到網路、權限不足、讀取逾時或瀏覽器擋住跨網域請求，可使用左側的「匯入 CSV」手動載入 UTF-8 CSV 檔案。

## v0.5 目前狀態

### 已完成且可使用

- 純前端靜態頁面，不需要安裝套件、登入或啟動後端服務。
- 自動讀取目前設定的 Google Sheet 第一個工作表（`gid=0`）CSV。
- 可手動匯入本機 UTF-8 CSV，作為 Google Sheet 無法讀取時的備援。
- 可依資料中的月份切換檢視，預設使用目前月份；若目前月份沒有資料，會選擇資料中最新月份。
- 可依欄位名稱而非欄位位置讀取資料，並支援常用中文欄位別名。
- 可自動解析金額、百分比、逗號與 `NT$` 等常見格式。
- 可計算本月分潤、目標差額、達成率、訂單、點擊、CTR、CVR 與 EPC。
- 若沒有直接提供實際分潤，可用「成交金額 × 佣金率」估算該列分潤。
- 若有商品價格與佣金率，可計算單筆預估分潤，供推薦與高分潤排序使用。
- 可依商品狀態與優先級自動補上基本的下一步行動。
- 商品名稱可連到貼文、分潤短連結或商品頁，優先順序為貼文 URL、分潤連結、商品連結。
- 可顯示本週推薦主推商品、低成效商品、高分潤商品與待處理清單。
- 圖表使用瀏覽器原生 Canvas 繪製，不依賴外部圖表套件。

### 目前可以協助處理的事項

- 每月目標追蹤：快速確認本月實際分潤與距離 NT$5,000 的差額。
- 基礎漏斗判讀：透過曝光、點擊、訂單、CTR、CVR 與 EPC 找出問題階段。
- 選品排序：優先找出高優先級、待發文及預估分潤較高的商品。
- 低成效提醒：標示「點擊達 10 次但尚未成交」、「曝光達 500 次但無點擊」，或曝光達 500 且 CTR 低於 1% 的商品。
- 高分潤提示：依單筆預估分潤或現有實際分潤由高至低排列。
- 待辦整理：集中列出仍需執行的商品，方便安排發文、測試或後續優化。
- 成效比較：依日期、品類及內容類型比較實際分潤與轉換表現。
- 資料連結跳轉：從商品名稱直接前往相關貼文、分潤連結或商品頁。

### 已知限制

- v0.5 實際上只讀取一個 Google Sheet CSV（目前固定為 `gid=0`），尚未自動合併「商品池、發文表、成效表」三張工作表。
- Google Sheet 必須能以 CSV 形式公開讀取；私人試算表無法直接由此純前端頁面安全存取。
- 匯入與整理只發生在瀏覽器記憶體中，重新整理後不會保存手動匯入的資料。
- Dashboard 目前只有讀取與分析功能，不能回寫 Google Sheet、修改狀態或直接建立發文排程。
- 月目標、資料網址與低成效門檻目前寫在程式中，尚未提供畫面設定。
- 系統不會跨列自動去重；同一筆資料若重複出現在 CSV，統計也會重複加總。
- 資料缺少日期、月份或核心成效欄位時，部分指標、圖表及提醒可能顯示為 0、空白或未分類。
- 目前沒有自動化測試、權限控管、多使用者與雲端資料庫。

## 建議 v0.6 可更新內容

建議優先順序如下：

1. 正式讀取並合併商品池、發文表、成效表三張工作表，以商品名稱、貼文 URL 或穩定 ID 建立關聯。
2. 加入設定面板，讓使用者調整 Google Sheet 網址、工作表 `gid`、每月目標與低成效門檻。
3. 加入資料驗證、重複資料偵測與匯入預覽，清楚標示缺少或格式錯誤的欄位。
4. 加入上月比較、週趨勢、日期範圍與商品／品類篩選，協助判斷成效變化。
5. 加入匯出月報或分享摘要功能，整理本月成果、問題商品與下一步行動。
6. 規劃安全的後端或 Google Sheets API 代理，支援私人試算表、狀態回寫與資料保存。
7. 補上自動化測試、鍵盤操作與無障礙檢查，降低後續版本更新造成的回歸問題。
