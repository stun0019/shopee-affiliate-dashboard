# Threads 發文核心

`content-engine.js` 是第一階段的純函式內容引擎，不讀取 Google Sheet，也不直接發布貼文。

## 輸入與輸出

`createThreadsDraft(product)` 接受商品名稱、賣點、發文角度、族群、風險、分潤連結與本週角色，輸出：

- `mainText`：2–4 行，單一鉤子，不含連結。
- `replyText`：一句自然承接、分潤連結與分潤揭露。
- `disclosure`：可獨立顯示的揭露文字。
- `contentType`：依角色、角度和族群推導的內容類型。
- `status`：固定從 `draft` 開始。

中文欄位和 camelCase 欄位都可以使用。風險只用於守門，不會被改寫成賣點。

## 文案原則

1. 首句像真人當下的情緒，不用制式商品介紹。
2. 第二句只講一個痛點、反差或具體用途，其餘資訊交給圖片或影片證明。
3. 有清楚族群與使用動作時，才使用「族群點名＋拜託式鉤子」；不讓每篇都使用同一個驚嘆句。
4. 家庭收納、重複拿取等商品優先採「誰設計的／終於有解」結構，並搭配整理前後或操作畫面。
5. 可愛、低價只能當輔助訊號，不能單獨當作高分享理由。
6. 夫妻或家人對話只能引用已確認的真實素材，不由系統捏造。
7. 熟悉角色或迷因商品必須另做 IP、仿品與素材授權審核。
8. 不捏造親身使用、價格、優惠、銷量與效果。像「續命 20 分鐘」這類數字效果，沒有可靠證據時會移除。

## 審核狀態

目前只允許依序前進：

```text
draft -> pending_review -> approved -> scheduled
```

進入 `approved` 前必須有有效的 HTTP(S) 分潤連結；跳級、倒退或未知狀態都會被拒絕。

## 測試

測試放在 `tests/content-engine.test.mjs`，使用 Node.js 內建測試工具：

```text
node --test tests/content-engine.test.mjs
```
