# 分潤工具 Dashboard 簡化規格給大冰奶

版本：`v0.5`

更新日期：`2026-07-16`

## 目標

建立「每月 NT$5,000 蝦皮分潤」的輕量 Dashboard，讓使用者可以在同一個畫面追蹤成果、判讀商品成效，並找到下一步要處理的商品。

此版本維持純前端、免登入、免安裝與可直接開啟的方向，不做複雜後台。

## 計算規則

- 資料月份：優先使用 `月份`；若未提供，從選品日期或一般日期取出 `YYYY-MM`；仍無法判斷時使用目前月份。
- 本月實際分潤：加總選定月份的實際分潤。
- 分潤備援估算：若實際分潤空白或無法解析，使用 `成交金額 × 佣金率`。
- 單筆預估分潤：優先使用預估分潤欄位，否則使用 `價格 × 佣金率`。
- 距離目標：`max(5000 - 本月實際分潤, 0)`。
- 達成率：`本月實際分潤 ÷ 5000`，進度條最高顯示 100%。
- CTR：若沒有提供可用值，使用 `點擊 ÷ 曝光`。
- CVR：若沒有提供可用值，使用 `訂單 ÷ 點擊`。
- EPC：KPI 使用 `本月實際分潤 ÷ 本月點擊`；單列未提供 EPC 時也使用該列實際分潤除以點擊。
- 所有除法在分母為 0 時顯示 `-` 或 0，不產生錯誤。

## 營運判斷規則

### 本週推薦主推商品

候選商品需有商品名稱，且至少符合待發文、有優先級、有預估分潤或已有實際分潤之一。

排序分數依序考量：

1. 優先級：高 > 中 > 低。
2. 待發文狀態。
3. 是否已有下一步行動。
4. 預估單筆分潤。
5. 實際分潤。
6. 點擊數。

畫面最多顯示前 4 筆。

### 低成效商品提醒

符合任一條件即列入：

- 點擊數大於等於 10，訂單數為 0。
- 曝光數大於等於 500，點擊數為 0。
- 曝光數大於等於 500，CTR 低於 1%。

畫面最多顯示前 4 筆。

### 高分潤商品提示

- 有預估單筆分潤或實際分潤的商品才列入。
- 優先依預估單筆分潤排序；沒有預估值時使用實際分潤。
- 畫面最多顯示前 4 筆。

### 待處理清單

- 排除含有未更新、觀察、淘汰、已發文、已完成、完成、停止主推的狀態。
- 其餘非空白狀態視為待處理。
- 依待發文、優先級、預估分潤、日期的順序排列。
- 若下次行動空白，依狀態或優先級推導：待發文 → 發文；有點擊 → 觀察轉單；成交 → 加碼／變體文；高優先級 → 優先測試。

## 商品連結規則

商品名稱的連結優先順序如下：

1. 貼文 URL。
2. 分潤連結。
3. 商品連結。

三者都沒有時只顯示商品名稱文字。

## UI 方向

- 第一眼要看到本月實際分潤、距離 NT$5,000 的差額與今日待處理事項。
- 以營運 Dashboard 為主，不做 landing page。
- 收益使用綠色，警示與待處理使用橘色，背景使用淺色。
- 手機版需能快速查看 KPI、推薦商品、警示與待辦，所有主要區塊改為單欄。
- 圖表由原生 Canvas 繪製，不引入外部圖表依賴。
